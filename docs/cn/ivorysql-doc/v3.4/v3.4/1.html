1. 版本概览

[发行日期：2024年9月26日]

IvorySQL 3.4基于PostgreSQL 16.4，并修复了多个问题。更多信息请参考 文档网站 。

2. 增强功能

  • PostgreSQL 16.4 的增强功能

    1. 在未经授权时防止pg_dump执行

    2. 避免归并右反连接计划产生错误结果

    3. 防止VACUUM中出现无限循环

    4. 修复在将表作为分区附加后失败的问题，前提是该表之前有继承子表

    5. 修复了 time 和 timetz 类型的 ISO-8601“扩展”时间格式的输入

    更多细节, 请参阅 PostgreSQL发布说明.

3. 问题修复

  • 新增 Oracle sequence功能: 问题 #588

  • 支持 xmlisvalid 函数: 问题 #695

  • 在Oracle模式下为字符类型限制设置为2000: 问题 #561

  • 支持使用SQL命令ALTER TABLE ADD (CONSTRAINT …​)向现有表添加约束: 问题 #612

  • 支持以下函数 TO_MULTI_BYTE(), TO_SINGLE_BYTE(), ASCIISTR(), COMPOSE(), DECOMPOSE(): 问题 #609

  • docker_library项目现已支持Debian的X86和ARM架构: #16

4. 源代码

IvorySQL主要包含2个代码仓库:

  • IvorySQL数据库源码: https://github.com/IvorySQL/IvorySQL

  • IvorySQL官方网站: https://github.com/IvorySQL/Ivory-www
